처방전 매핑 로직 문서¶

개요¶

본 문서는 17개 사용자 표현용 측정항목 점수를 기반으로 14개 처방 항목(A01-A14)으로 매핑하고, 이를 다시 믹스 코드로 변환하여 처방전을 생성하는 전체 로직을 설명합니다.

매핑 규칙¶

  1. 다대일 매핑: 하나의 처방 항목에 여러 측정항목이 매핑될 수 있음

  2. 예: age_spot ← melasma_score, freckle_score, post_acne_pigment_score

  3. 최소 점수 기준: 매핑된 측정항목 중 가장 낮은 점수를 기준으로 처방 비율 결정

  4. 이유: 가장 취약한 부분을 집중 케어하기 위함

  5. 일대일 매핑: 하나의 측정항목이 하나의 처방 항목에만 매핑됨

  6. 예: dullness_score → A01 (광채)

2. 점수 → 처방 비율 변환¶

변환 규칙¶

점수 범위 등급 처방 비율 설명
76-100점 Good 0% 처방 없음
41-75점 Moderate 1.0% ~ 3.0% 선형 계산
0-40점 Critical 3.0% 최대 처방

선형 계산 공식 (41-75점)¶

percentage = max_percentage - ((score - critical_threshold) * (max_percentage - min_percentage) / (good_threshold - critical_threshold))

기본 파라미터: - good_threshold: 76 - critical_threshold: 40 - max_percentage: 3.0 - min_percentage: 1.0

계산 예시: - 41점: 3.0 - ((41 - 40) * 2.0 / 34) = 3.0 - 0.059 = 2.94 → 2.9% - 60점: 3.0 - ((60 - 40) * 2.0 / 34) = 3.0 - 1.176 = 1.82 → 1.8% - 75점: 3.0 - ((75 - 40) * 2.0 / 34) = 3.0 - 2.059 = 0.94 → 1.0%

특징: - 0.1% 단위로 반올림 - 범위 제한: 1.0% ~ 3.0% - 점수가 낮을수록 처방 비율 높음

llm_prompt_template.md¶

  • PRESCRIPTION_CONFIG 섹션: 매핑 테이블, 변환 규칙 문서화
  • 목적: 개발자/사용자에게 매핑 구조 제공

config.json¶

  • prescription.measurement_to_prescription_mapping: 17개 측정항목 → 처방 항목 매핑
  • prescription.prescription_item_to_mix_code: 처방 항목 → 믹스 코드 매핑
  • prescription.skin_assessment: 점수 → 처방 비율 변환 파라미터
  • good_threshold: 76
  • critical_threshold: 40
  • max_percentage: 3.0
  • min_percentage: 1.0

prescription_calculator.py¶

  • SKIN_ASSESSMENT_TO_PRESCRIPTION_MAPPING_DEFAULT: 기본 매핑 (config 로드 실패 시)
  • PRESCRIPTION_ITEM_TO_MIX_CODE_DEFAULT: 기본 믹스 코드 매핑
  • _get_measurement_to_prescription_mapping(): config.json 동적 로드
  • _get_prescription_item_to_mix_code(): config.json 동적 로드
  • calculate_skin_assessment_recipe(): 전체 처방전 계산

6. 유연성 및 확장성¶

매핑 변경¶

  • 새로운 측정항목 추가: config.json에 매핑 추가
  • 기존 매핑 수정: config.json 수정
  • 처방 항목 추가: config.json에 처방 항목 및 믹스 코드 추가

파라미터 변경¶

  • 점수 임계값 변경: config.json의 good_threshold, critical_threshold 수정
  • 처방 비율 변경: config.json의 max_percentage, min_percentage 수정

확장 예시¶

새로운 측정항목 추가 (예: 새로운 직교 항목):

  1. llm_prompt_template.md에 매핑 추가
  2. config.json에 매핑 추가
  3. 코드 수정 불필요 (자동 반영)

7. 주의사항¶

  1. 매핑 일관성: llm_prompt_template.md와 config.json의 매핑이 일치해야 함
  2. 점수 범위: 모든 점수는 0-100 범위여야 함
  3. 처방 비율 합계: 처방 비율 합계에 제한 없음 (각 항목 독립 계산)
  4. 미사용 항목: 현재 미사용 항목은 향후 확장을 위해 예약됨
  5. config reload: 매핑 변경 후 config reload 필요
